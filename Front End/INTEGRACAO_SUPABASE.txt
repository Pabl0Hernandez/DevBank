🎯 PRÓXIMOS PASSOS (Na Ordem):
═════════════════════════════════════════════════════════════════════════════

1️⃣  CRIAR ARQUIVO .env
    ┌─────────────────────────────────────────────────────────────────┐
    │ cp .env.example .env                                            │
    │ Editar com suas credenciais Supabase                            │
    │ VITE_SUPABASE_URL=https://seu-projeto.supabase.co              │
    │ VITE_SUPABASE_ANON_KEY=sua-chave-anon-publica                  │
    └─────────────────────────────────────────────────────────────────┘

2️⃣  EXECUTAR SQL NO SUPABASE
    ┌─────────────────────────────────────────────────────────────────┐
    │ Supabase → SQL Editor → New Query                               │
    │ Copiar todo conteúdo de: database.sql                           │
    │ Colar no editor → Run                                           │
    │ ✓ Tabelas: cards, transactions, banks                           │
    │ ✓ RLS habilitado                                                │
    │ ✓ Políticas criadas                                             │
    │ ✓ 8 bancos inseridos                                            │
    └─────────────────────────────────────────────────────────────────┘

3️⃣  INSTALAR DEPENDÊNCIAS
    ┌─────────────────────────────────────────────────────────────────┐
    │ npm install                                                     │
    │ Aguardar conclusão sem erros                                    │
    └─────────────────────────────────────────────────────────────────┘

4️⃣  INICIAR SERVIDOR
    ┌─────────────────────────────────────────────────────────────────┐
    │ npm run dev                                                     │
    │ Abrir: http://localhost:5173                                    │
    └─────────────────────────────────────────────────────────────────┘

5️⃣  CRIAR USUÁRIO DE TESTE
    ┌─────────────────────────────────────────────────────────────────┐
    │ Clicar em "Registrar"                                           │
    │ Preencher: nome, email, senha                                   │
    │ Confirmar login bem-sucedido                                    │
    └─────────────────────────────────────────────────────────────────┘

6️⃣  INSERIR DADOS DE TESTE (Opcional)
    ┌─────────────────────────────────────────────────────────────────┐
    │ Supabase → SQL Editor → New Query                               │
    │ Copiar: database-seed.sql                                       │
    │ Substituir 'seu-user-id-aqui' pelo seu UUID real               │
    │ Run → Verificar dados em Dashboard                              │
    └─────────────────────────────────────────────────────────────────┘

🔐 TABELAS CRIADAS:
═════════════════════════════════════════════════════════════════════════════

💳 CARDS
   ├─ id (PK)
   ├─ user_id (FK → auth.users)
   ├─ holder, number, expiry, cvv
   ├─ brand, balance, limit
   ├─ color, bank, type
   └─ RLS: Apenas proprietário pode acessar

📊 TRANSACTIONS
   ├─ id (PK)
   ├─ user_id (FK → auth.users)
   ├─ name, category, amount, date
   ├─ icon, color, type, card_id
   └─ RLS: Apenas proprietário pode acessar

🏦 BANKS
   ├─ id (PK)
   ├─ name, sub, color, logo
   ├─ connected
   └─ RLS: Qualquer autenticado pode ver (dados públicos)

❓ DÚVIDAS FREQUENTES:
═════════════════════════════════════════════════════════════════════════════

P: Onde coloco meu .env?
R: Em Front End/.env (raiz da pasta, não em src/)

P: Qual o formato do VITE_SUPABASE_URL?
R: https://seu-projeto.supabase.co (sem barra final)

P: Como encontro meu user_id?
R: Supabase → Authentication → Users → clique no usuário → copie ID

P: Posso usar dados reais de cartão?
R: NÃO! Use dados fictícios apenas para testes.

P: Como faço para adicionar mais campos?
R: Veja DATABASE_SCHEMA.md → Extensões futuras

P: Preciso de backend? Ou é tudo via Supabase?
R: Tudo via Supabase! Nenhum backend próprio necessário.
