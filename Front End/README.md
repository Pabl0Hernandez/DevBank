# DevBank Frontend

Este projeto é a parte de frontend do DevBank. Agora ele está configurado para usar o Supabase como backend sem você precisar criar uma API própria.

## Instalação

1. Copie `.env.example` para `.env`
2. Preencha as variáveis:

```env
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-public-key
```

3. Instale dependências:

```bash
npm install
```

4. Rode em desenvolvimento:

```bash
npm run dev
```

## Integração com Supabase

O frontend agora usa:

- `src/services/supabaseClient.js` para se conectar ao Supabase
- `src/services/supabaseApi.js` para autenticação, cartões, transações, bancos e estatísticas

### Tabelas necessárias no Supabase

Crie estas tabelas no editor SQL do Supabase ou no SQL Editor:

```sql
create table cards (
  id bigint generated by default as identity primary key,
  user_id uuid references auth.users(id),
  holder text,
  number text,
  expiry text,
  cvv text,
  brand text,
  balance numeric,
  limit numeric,
  color text,
  bank text,
  type text
);

create table transactions (
  id bigint generated by default as identity primary key,
  user_id uuid references auth.users(id),
  name text,
  category text,
  amount numeric,
  date date,
  icon text,
  color text,
  type text,
  card_id bigint references cards(id)
);

create table banks (
  id bigint generated by default as identity primary key,
  name text,
  sub text,
  color text,
  logo text,
  connected boolean default false
);
```

### Permissões básicas

No Supabase, habilite `select`, `insert`, `update`, `delete` para `authenticated` (ou configure políticas RLS específicas) nas tabelas `cards`, `transactions` e `banks`.

## Uso do backend sem API própria

- Autenticação: `auth.signInWithPassword` e `auth.signUp`
- Dados: `cards`, `transactions`, `banks` armazenados diretamente no Supabase
- Estatísticas: calculadas no frontend a partir das transações do usuário

## Observações

Este projeto ainda mantém o estilo atual e substitui apenas o backend fake por Supabase. Se quiser avançar, crie também `profiles` para campos extras do usuário e `contacts` para o envio de dinheiro.
